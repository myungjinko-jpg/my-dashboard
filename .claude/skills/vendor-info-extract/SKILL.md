# 거래처 정보 추출 (Vendor Info Extract)

퍼블리셔 내부에서 신규 파트너사를 **거래처로 등록**할 때, 법인등록증·법인통장 서류를 읽어
계약 관리 탭의 "거래처 정보" 폼 8개 필드에 그대로 붙여넣을 수 있는 텍스트로 정리한다.

이 작업의 핵심 가치는 **정확성**이다. 거래처 등록 정보가 틀리면 지급·정산이 막히므로,
**문서에 실제로 적힌 값만** 옮기고 없는 값은 절대 지어내지 않는다.

## 0. 입력 확인

스킬이 호출됐는데 **첨부된 서류가 없으면**, 값을 지어내지 말고 먼저 이렇게 안내한다:

> 거래처 정보를 추출하려면 **법인등록증과 법인통장을 업로드해주세요.**
> (법인등록증 → 식별번호·법인명·국가·주소·대표 / 법인통장 → 계좌번호)

서류가 첨부되면 진행한다.

1. **법인등록증**(필수): 사업자등록증 / Business Registration Certificate / Certificate of Incorporation 등.
   → 거래처식별번호·거래처명·국가·주소·대표를 여기서 읽는다.
2. **법인통장**(권장): 통장 사본 / Bank statement 표지.
   → 거래처계좌번호를 여기서 읽고, **예금주명이 법인명과 동일한지** 확인한다.

문서가 이미지·PDF·스캔본 어느 형태든 그대로 읽는다. 여러 장이면 모두 확인한다.

**거래처담당자는 서류에서 뽑지 않는다** — 이 값은 우리 내부의 담당 PM이라 법인등록증·통장에 없다.
출력에서 담당자 필드는 제외한다(대시보드에서 파트너사 담당자로 자동 입력됨).

## 1. 필드별 추출 규칙

| 폼 필드 | 출처 | 추출 지침 |
|---|---|---|
| **거래처식별번호** | 법인등록증 | 법인등록번호 / 사업자등록번호 / Business Registration No. / Tax ID / Company Number 등 문서에 기재된 식별번호. 하이픈 포함 원문 그대로. |
| **거래처명** | 법인등록증 | 법인명(상호). 영문 문서면 영문 법인명 그대로(Co., Ltd. 등 접미사 포함). |
| **거래처국가** | 법인등록증 | **법인등록증에 기재된 법인 소재지(등록 관할) 기준.** `Country (KST±N)` 형식. 아래 [국가·KST 표기] 참조. |
| **거래처주소** | 법인등록증 | 본점 소재지 전체 주소(도시·우편번호 포함). 원문 언어 그대로. |
| **거래처대표** | 법인등록증 | 대표자 / Representative / Director 성명. |
| **거래처Email** | 서류에 있으면 | 서류에 명시돼 있으면 기재, 없으면 빈칸 + `(문서에 없음 — 수동 확인)`. |
| **거래처계좌번호** | 법인통장 | 계좌번호 원문 그대로. 통장이 없으면 빈칸 + `(법인통장 미제공)`. |

> **거래처담당자**는 출력하지 않는다 — 내부 담당 PM이라 서류에 없고, 대시보드에서 파트너사 담당자로 자동 채워진다.

최종적으로 서류에서 뽑는 정보는 **7개**: 거래처식별번호 / 거래처명 / 거래처국가 / 거래처주소 / 거래처대표 / 거래처Email / 거래처계좌번호.

### 언어 정책 (국내=한글, 해외=영어)

값의 표기 언어는 **발행국 기준**으로 통일한다.

- **국내(한국) 서류**: 상호·대표자·주소를 **한글 원문 그대로**. (등록 시 한글이 정본)
- **해외 서류**: 법인명·대표자·주소를 **모두 영어로**.
  - 영문이 병기된 서류면 그 영문 표기를 그대로 쓴다.
  - 현지어(베트남어·태국어·터키어 등)만 있으면 표준 로마자/영문으로 옮겨 적고, 뒤에 `(현지어 원문에서 영문 변환)`을 한 줄 남긴다.
  - 단, **식별번호·계좌번호처럼 숫자·코드값은 언어와 무관하게 원문 그대로** 둔다 (번역·변형 금지).

`거래처국가`는 국내·해외 모두 항상 영문 `Country (KST±N)` 형식이다.

### 추정 금지 (가장 중요)

- 문서에 **명확히 적혀 있는 값만** 옮긴다.
- 흐릿하거나 판독이 애매하면 지어내지 말고 `(판독 불가 — 원본 확인 필요)`로 표기한다.
- 담당자·Email처럼 서류에 없는 항목은 **비워두고 표시**한다. 그럴듯한 값을 만들어내는 것은 등록 오류로 직결되므로 절대 하지 않는다.

## 2. 국가·KST 표기

`거래처국가`는 파트너와의 커뮤니케이션 시차 파악용으로 **`Country (KST±N)`** 형식으로 적는다.
`N` = 해당 국가 표준시 − 한국 표준시(KST, UTC+9). 즉 한국이 기준(0).

예시:
- Korea → `Korea (KST+0)`
- Vietnam / Thailand (UTC+7) → `Vietnam (KST-2)`
- Turkey (UTC+3) → `Turkey (KST-6)`
- China / Singapore (UTC+8) → `China (KST-1)`
- Japan (UTC+9) → `Japan (KST+0)`
- India (UTC+5:30) → `India (KST-3:30)`
- UK (UTC+0) → `UK (KST-9)`
- 미국 서부(UTC-8) → `USA (KST-17)` (주별 시차가 있으면 도시/주도 함께: `USA-CA (KST-17)`)

여름시간(DST)이 있는 국가는 표준시 기준으로 적고, 애매하면 `(DST 변동)`을 덧붙인다.
국가가 여러 시간대를 가지면(미국 등) 법인등록증 주소의 도시 기준으로 판단하고 근거를 한 줄 남긴다.

### 등록 국가 = 법인등록증 기준 (중요)

거래처국가는 **법인등록증에 적힌 법인 소재지(설립·등록 관할)**로 정한다.
개발사가 실제로 활동하는 곳(개발 스튜디오 위치)과 **법인이 등록된 국가가 다른 경우가 있다**
(예: 개발은 베트남에서 하지만 법인은 싱가포르에 설립). 지급·정산은 법인 기준으로 이뤄지므로
**항상 법인등록증에 기재된 국가**를 거래처국가로 쓴다.

문서에 등록 관할(발행국)이 별도로 명시돼 있고 그것이 주소 국가와 다르면, 등록 관할을 우선하고
확인 메모에 `법인 등록국 X ≠ 주소상 소재 Y` 로 남긴다. 스튜디오의 실제 개발 위치가 다르다는
정황이 있어도 **거래처국가는 법인등록증 기준을 유지**하고, 다른 점은 메모로만 표기한다.

참고: 대시보드에는 별도로 **개발소재지**(우리가 소통하는 팀 위치, 사이드바·시차 기준) 필드가 있다.
이건 우리 내부 정보라 서류에서 뽑지 않으며, 스킬이 출력하는 `거래처국가`는 어디까지나 **법인 등록국**이다.
개발 위치가 다르면 확인 메모에 한 줄 남겨 대시보드에서 개발소재지를 따로 지정하도록 안내한다.

## 3. 출력 형식

아래 형식을 **정확히** 사용한다. 사용자가 대시보드 폼에 그대로 옮겨 적을 수 있도록,
폼 필드명과 1:1로 맞춘 라벨:값 블록으로 낸다.

```
거래처식별번호: <값>
거래처명: <값>
거래처국가: <Country (KST±N)>
거래처주소: <값>
거래처대표: <값>
거래처Email: <값 또는 (문서에 없음 — 수동 확인)>
거래처계좌번호: <값 또는 (법인통장 미제공)>
```
(거래처담당자는 대시보드에서 파트너사 담당 PM으로 자동 입력되므로 여기서 출력하지 않는다.)

블록 아래에 **확인 메모**를 짧게 덧붙인다:
- 예금주명 ↔ 법인명 일치 여부 (통장 제공 시)
- 판독 불가·수동 확인 필요 항목 요약
- 국가 KST 판단 근거(다시간대 국가일 때만)

### 예시

**입력**: 베트남 게임 스튜디오의 영문 Business Registration Certificate + 법인통장 표지

**출력**:
```
거래처식별번호: 0316xxxxxx
거래처명: ABC Games Co., Ltd.
거래처국가: Vietnam (KST-2)
거래처주소: 12 Nguyen Hue, District 1, Ho Chi Minh City, 700000, Vietnam
거래처대표: Nguyen Van A
거래처Email: (문서에 없음 — 수동 확인)
거래처계좌번호: 1234567890 (Vietcombank)
```
확인 메모:
- 예금주명 "ABC Games Co., Ltd." = 법인명 일치 ✅
- Email은 서류에 없어 비워둠 — 스튜디오에 별도 요청 필요
- Vietnam 표준시 UTC+7 → KST(UTC+9) 대비 -2시간

**입력**: 국내 스튜디오의 사업자등록증(한글)

**출력**:
```
거래처식별번호: 123-45-67890
거래처명: (주)넵튠게임즈
거래처국가: Korea (KST+0)
거래처주소: 서울특별시 강남구 테헤란로 123, 4층
거래처대표: 홍길동
거래처Email: (문서에 없음 — 수동 확인)
거래처계좌번호: (법인통장 미제공)
```
확인 메모:
- 국내 서류이므로 상호·대표·주소 한글 원문 유지
- 법인통장 미제공 — 계좌번호 별도 확인 필요

## 4. 마무리

출력 후, 사용자가 계약 관리 탭 → 해당 파트너 **거래처등록** 항목을 열어
"거래처 정보" 섹션에 붙여넣으면 된다고 한 줄 안내한다.
(현재는 수동 붙여넣기 단계이며, 추후 API 연동 시 자동 입력으로 확장 예정.)
