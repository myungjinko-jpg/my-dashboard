# 계약 관리 탭 — 기획 (PROCESS.md)

> 이 탭의 역할 정의와 프로세스 규칙. 기능 결정이 흔들릴 때 여기로 돌아온다.

## 한 줄 정의

**파트너십 행정 프로세스 러너** — "지금 뭘 해야 하고, 어디가 막혀 있는지"를 보여준다. 기록의 원장(源帳)은 Notion DB이고, 대시보드는 다음 액션을 안내하는 곳이다.

## 사용자

- **주 사용자**: biz/PM (본인) — 매일 열어 다음 액션 확인·처리
- **부 사용자**: biz 팀 — Slack `#biz_alert` 알림으로 현황 수신

## 원칙

1. **대시보드 = 프로세스 러너, Notion = 원장.** 완전한 기록 조회·정정은 Notion에서. 대시보드에 필드를 늘리는 것보다 "다음 할 일"이 명확한 게 우선.
2. 모든 화면 요소는 셋 중 하나에 답해야 한다: **다음 할 일은? / 막힌 곳은? / 만료 임박은?**
3. **같은 정보는 한 곳에서만 입력.** 파트너 공통 서류(법인등록증·법인통장)=거래처등록에서, 첫 프로젝트 부속합의서=파트너십계약에 포함, 국가=거래처등록의 거래처국가.

## 데이터 모델

- **파트너 공통 단계(순차)**: 파트너십계약 → (NDA, 선택) → 거래처등록
- **프로젝트별 단계(순차, 프로젝트마다 반복)**: 부속합의서 → 지출기안(프로토타입 → 이터레이션#N 반복)
- **상태**: 요청전 → 진행중 → 완료 (클릭 순환)
- **국가**: 거래처등록의 `거래처국가` 필드가 원본. **국문 표기로 통일** (예: 한국, 일본, 중국, 미국). 사이드바 분류·표시·소팅에 사용.
- **자동갱신**: 파트너십계약 등 계약류의 자동갱신 조항(예: Minder 계약 Article 7, 180일 자동갱신). 체크 시 만료 경보 대상에서 제외.

## 핵심 시나리오

### 1. 신규 파트너 온보딩
1. `+ 파트너사`: 파트너사명 + 첫 프로젝트명(선택) + 국가(선택) 입력
2. 자동 생성: 파트너십계약(요청전) → 거래처등록(요청전, 국가 포함) → 부속합의서(완료·파트너십계약 포함) → 프로토타입 지출기안(요청전)
3. 순차 아코디언으로 파트너십계약부터 진행 (소프트 잠금 — 필요 시 순서 건너뛰기 허용)

### 2. 프로젝트 시작 (2번째 프로젝트부터)
1. `+ 프로젝트`: 부속합의서(요청전, 별도 체결) → 프로토타입 지출기안(요청전) 생성
2. 부속합의서 체결 → 계약서 URL 등록 → 지출기안 진행

### 3. 이터레이션 정산 (반복)
1. 이터레이션 개발 확정 → `+ 지출기안` (제목 자동: 이터레이션#N)
2. 서류 확인: 법인등록증·법인통장(거래처등록 연동) + 부속합의서·스펙내용·인보이스(프로젝트별 드라이브 링크)
3. 해외 송금 정보 확인 → 네이버웍스 기안 상신 → 기안 링크 등록 → 완료

## 단계별 완료 조건

완료 처리는 막지 않되, 조건 미충족 시 **경고 표시**한다(soft). — [확정: 경고 방식]

| 단계 | 완료 조건 |
|------|-----------|
| 파트너십계약 / NDA | 계약서 URL 등록 |
| 거래처등록 | 서류 2종(링크 또는 체크) + 거래처식별번호·거래처명 |
| 부속합의서 | 계약서 URL 등록 (파트너십계약 포함이면 면제) |
| 지출기안 | 기안 링크 등록 |

## 알림 규칙 (Slack `#biz_alert`)

- **만료 임박**: 계약류(파트너십계약·부속합의서·NDA) 만료일 D-30 이내. **자동갱신 계약 제외.**
- **서류 미비**: 진행중 항목의 미수령 서류. 파트너 공통 서류는 거래처등록 상태 연동.
- **[예정] 정체 감지**: `최종업데이트일` 기준 7일 이상 무변화인 요청전·진행중 항목.
